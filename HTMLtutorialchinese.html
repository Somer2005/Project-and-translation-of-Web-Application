开头部分的处理：关于 <!DOCTYPE> 以及后面的内容，包括 <head> 部分的处理

在正式开始写页面内容之前，HTML 文档本身也需要一个“合法的开头”。 这个开头并不会直接显示在页面上，但它决定了浏览器应当 “以什么规则来理解后面的所有内容”。

第一行通常是 <!DOCTYPE> 声明。

<!DOCTYPE> 并不是一个 HTML 标签， 而是一条文档类型声明， 它的作用是告诉浏览器：当前文档使用的是哪一种 HTML 规范。 


<!DOCTYPE html>

在现代 Web 开发中，这一行几乎是固定写法， 表示使用 HTML5 标准。 如果省略这一行，浏览器可能会进入“兼容模式”， 从而导致一些布局和样式行为变得不可预测。

在 <!DOCTYPE> 之后，紧接着是 <html> 标签， 它是整个 HTML 文档的根元素。 通常会在这里声明页面的主要语言， 这对搜索引擎和无障碍工具都非常重要。 


<html lang="zh-CN">

接下来是 <head> 部分。 <head> 中的内容不会直接显示在页面上， 但它们定义了页面的元信息、编码方式、标题以及外部资源的引入方式。

在 <head> 中最重要的一项，是字符编码声明。 


<meta charset="UTF-8">

这一行用于声明文档使用 UTF-8 编码， 如果缺失或设置错误，中文内容很容易出现乱码问题。

另一个几乎必不可少的是视口设置， 它主要用于控制页面在移动设备上的显示方式。 


<meta name="viewport" content="width=device-width, initial-scale=1.0">

页面的标题通过 <title> 标签指定， 它不会显示在正文中， 但会显示在浏览器标签页、收藏夹以及搜索结果中。 


<title>一个使用 HTML 实现的常见 HTML 代码表</title>

此外，<head> 中还常用于引入外部资源， 例如 CSS 样式表。 


<link rel="stylesheet" type="text/css" href="stylecss.css">

总体来说， <head> 负责回答的是“这个页面是什么、该如何被解析”， 而不是“这个页面展示了什么内容”。 真正可见的页面结构与文本， 则全部放在后面的 <body> 中。

第二部分 代码块的展示方法

在编写教程文档或博客时，经常需要展示代码块。HTML 提供了多种方式来呈现代码：

1. 使用 <pre> 和 <code> 标签

这是最语义化的方式，用来展示代码或预格式化文本（保留所有空格和换行）：

代码示例：

<pre>
  <code>
    function hello() {
      console.log('Hello World');
    }
  </code>
</pre>

实际效果：

function hello() {
  console.log('Hello World');
}

2. 使用 <pre> 标签单独

<pre> 标签可以单独使用，它会保留文本的所有空格和换行。不加 <code> 也能正常工作：

效果：

const arr = [1, 2, 3];
arr.forEach(item => {
  console.log(item);
});

3. 使用 <code> 标签（内联代码）

用于在段落中展示短代码片段。例如：变量名 userName 或方法 parseInt()。

4. 结合 CSS 美化代码块

为了更好的阅读体验，可以用 CSS 添加背景色、边框和字体：

pre {
  background-color: #f5f5f5;
  border: 1px solid #ddd;
  padding: 10px;
  border-radius: 4px;
  overflow-x: auto;
}
code {
  font-family: 'Courier New', monospace;
  font-size: 14px;
}

那么我们来解决最后一个问题：既然<prt>就有作用，那我为什么还要用<code>呢？

实际上，这是一个“声明符”，声明符是什么意思？就是类似注释的作用。我们知道HTML在Web开发中扮演的角色是回答“这是什么”的问题的，对肉眼来说没有区别，但是这对搜索引擎和翻译器会很友好。

第三部分 <div>、<section>、<article> 与 <nav> 的区别与应用场景

在写 HTML 的时候，最容易让人迷惑的并不是语法，而是“我现在该用哪个标签”。 尤其是 <div>、<section>、<article> 和 <nav>， 它们在页面显示效果上几乎没有任何区别，但在语义层面上差异非常大。

首先从最基础、也是最容易被滥用的 <div> 说起。

<div> 是一个完全没有语义的通用容器。 它的唯一作用，就是“把一坨东西包起来”。 如果你只是为了布局、加样式、方便 JavaScript 操作， 而这部分内容本身并不表达任何文档结构含义，那么用 <div> 是合理的。 


<div class="container">
    <div class="left">左侧内容</div>
    <div class="right">右侧内容</div>
</div>

接下来是 <section>。 <section> 用来表示文档中的一个逻辑章节， 通常应该包含一个标题。 它强调的是“这一部分在讲同一个主题”， 但并不要求这部分内容可以脱离上下文独立存在。 


<section>
    <h2>安装说明</h2>
    <p>这里描述软件的安装步骤。</p>
</section>

<article> 则更进一步。 它表示的是一段语义上完整、可独立存在的内容。 即使把这段内容单独拿出去， 读者依然能理解它在讲什么。 


<article>
    <h2>Linux 配置迁移指南</h2>
    <p>本文介绍如何完整迁移一台 Linux 设备的配置。</p>
</article>

最后是 <nav>。 <nav> 专门用于包裹导航相关的链接集合， 它关注的不是内容本身，而是页面或文档之间的跳转关系。 


<nav>
    <ul>
        <li><a href="#part1">第一部分</a></li>
        <li><a href="#part2">第二部分</a></li>
        <li><a href="#part3">第三部分</a></li>
    </ul>
</nav>

简单总结一下： <div> 只是容器； <section> 是文档中的章节； <article> 是可以独立存在的内容单元； 而 <nav> 则只用于导航。 在不确定的时候，优先考虑是否能赋予语义， 只有在“确实没有语义”的情况下，才使用 <div>。
